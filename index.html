How it works

Three steps to your
own SMS infrastructure

No carrier contracts. No SLA negotiations. Deploy in minutes and start sending.

01
Deploy the backend
Run the setup script. It creates your Cloudflare D1 database, applies all migrations, generates a random JWT secret, and deploys the Worker — completely automated.
bash setup.sh
02
Pair Android devices
Install the SMS Flare app on any Android phone. Scan the QR code from your dashboard — it encodes both the pairing token and API URL so you don't type a thing.
Download APK
03
Send via API or dashboard
POST to the REST API using an API key or your session JWT. Jobs are assigned to available devices within seconds and dispatched over the device's SIM card.
POST /api/sms/send

Features

Everything you need,
nothing you don't

Built for developers who want control without the overhead of a managed SMS provider.

Zero carrier fees
Use your existing SIM plan. The only cost is the phone and its data plan — no per-message markup, no monthly minimums, no surprise bills.
Full infrastructure ownership
Cloudflare Workers + D1 SQLite. Your data never touches a third-party SMS provider's servers. GDPR-compliant by architecture.
Clean REST API
Authenticate with a user JWT or a long-lived API key. Works with any language or HTTP client. Full response schemas documented.
Real-time delivery tracking
Every job has a full status timeline: pending → assigned → sent → delivered. Check the dashboard or poll the API for live updates.
Multi-device pool
Register 100+ Android devices. Jobs are assigned to whichever device sent the most recent heartbeat. Add more devices at any time without downtime.
Batch sending
POST up to 50 messages in one request. All validated before any job is created — a single invalid number rejects the whole batch cleanly.
REST API

One endpoint
to send anything

Three auth modes — user JWT, long-lived API key, or device token for Android. One consistent response shape. No SDK required.
